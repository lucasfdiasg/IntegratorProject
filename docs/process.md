# Processo de Trabalho e Governança

* **Ciclo de Trabalho:** Semanal (Sprints curtas focadas nas entregas do cronograma do PI).
* **Fluxo de Branches:** Utilização de GitHub Flow (`main` protegida, desenvolvimento em branches `feature/nome-da-tarefa`).
* **Regra de Pull Request (PR):** Nenhum código ou documento entra na `main` sem um PR. O PR exige aprovação (Review) de pelo menos 1 membro diferente do autor.

## Definition of Ready (DoR) - Quando podemos começar:
1. O requisito/tarefa está claro, documentado na SRS e não possui dependências bloqueantes de hardware/software.
2. A métrica de sucesso (Critério de Aceitação) é conhecida.

## Definition of Done (DoD) - Quando consideramos pronto:
1. O código foi implementado ou o circuito montado.
2. O Critério de Aceitação descrito na SRS foi validado em teste de bancada ou software.
3. O Pull Request foi revisado e mergeado.
